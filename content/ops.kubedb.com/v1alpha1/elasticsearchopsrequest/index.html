ElasticsearchCustomConfigurationSpec

ElasticsearchCustomConfigurationSpec is the spec for Reconfiguring the Elasticsearch Settings

Field Description
configSecret
Kubernetes core/v1.LocalObjectReference 		(Optional)

ConfigSecret is an optional field to provide custom configuration file for database.
secureConfigSecret
Kubernetes core/v1.LocalObjectReference 		(Optional)

SecureConfigSecret is an optional field to provide secure settings for database. - Ref: https://www.elastic.co/guide/en/elasticsearch/reference/7.14/secure-settings.html
applyConfig
map[string]string 		(Optional)

ApplyConfig is an optional field to provide Elasticsearch configuration. Provided configuration will be applied to config files stored in ConfigSecret. If the ConfigSecret is missing, the operator will create a new k8s secret by the following naming convention: {db-name}-user-config . Expected input format: applyConfig: file-name.yml: | key: value elasticsearch.yml: | thread_pool: write: size: 30
removeCustomConfig
bool 		(Optional)

If set to "true", the user provided configuration will be removed. The Elasticsearch cluster will start will default configuration that is generated by the operator.
removeSecureCustomConfig
bool 		(Optional)

If set to "true", the user provided secure settings will be removed. The elasticsearch.keystore will start will default password (i.e. "").
