CoordinatorSpec

(Appears on:MariaDBSpec, MongoDBSpec, MySQLSpec, PostgresSpec, RedisSpec)

CoordinatorSpec defines attributes of the coordinator container

Field Description
resources
Kubernetes core/v1.ResourceRequirements 		(Optional)

Compute Resources required by coordinator container. Cannot be updated. More info: https://kubernetes.io/docs/concepts/configuration/manage-compute-resources-container/
securityContext
Kubernetes core/v1.SecurityContext 		(Optional)

Security options the coordinator container should run with. More info: https://kubernetes.io/docs/concepts/policy/security-context/ More info: https://kubernetes.io/docs/tasks/configure-pod-container/security-context/

DatabasePhase (string alias)

(Appears on:ElasticsearchStatus, EtcdStatus, MariaDBStatus, MemcachedStatus, MongoDBStatus, MySQLStatus, PerconaXtraDBStatus, PgBouncerStatus, PostgresStatus, ProxySQLStatus, RedisSentinelStatus, RedisStatus, MongoDBOverviewSpec)

Value Description

"Critical"

used for Databases that can connect, ReplicaReady == false || Ready == false (eg, ES yellow)

"DataRestoring"

used for Databases for which data is currently restoring

"Halted"

used for Databases that are halted

"NotReady"

used for Databases that can't connect

"Provisioning"

used for Databases that are currently provisioning

"Ready"

used for Databases that are currently ReplicaReady, AcceptingConnection and Ready

TerminationPolicy (string alias)

(Appears on:ElasticsearchSpec, EtcdSpec, MariaDBSpec, MemcachedSpec, MongoDBSpec, MySQLSpec, PerconaXtraDBSpec, PostgresSpec, RedisSentinelSpec, RedisSpec)

Value Description

"Delete"

Deletes database pods, service, pvcs but leave the stash backup data intact.

"DoNotTerminate"

Rejects attempt to delete database using ValidationWebhook.

"Halt"

Deletes database pods, service but leave the PVCs and stash backup data intact.

"WipeOut"

Deletes database pods, service, pvcs and stash backup data.
