ClusterAuthMode (string alias)

(Appears on:MongoDBSpec)

ClusterAuthMode represents the clusterAuthMode of mongodb clusters ( replicaset or sharding) ref: https://docs.mongodb.com/manual/reference/program/mongod/#cmdoption-mongod-clusterauthmode

Value Description

"keyFile"

ClusterAuthModeKeyFile represents keyFile mongodb clusterAuthMode. In this mode, Use a keyfile for authentication. Accept only keyfiles.

"sendKeyFile"

ClusterAuthModeSendKeyFile represents sendKeyFile mongodb clusterAuthMode. This mode is for rolling upgrade purposes. Send a keyfile for authentication but can accept both keyfiles and x.509 certificates.

"sendX509"

ClusterAuthModeSendX509 represents sendx509 mongodb clusterAuthMode. This mode is usually for rolling upgrade purposes. Send the x.509 certificate for authentication but can accept both keyfiles and x.509 certificates.

"x509"

ClusterAuthModeX509 represents x509 mongodb clusterAuthMode. This is the recommended clusterAuthMode. Send the x.509 certificate for authentication and accept only x.509 certificates.

SSLMode (string alias)

(Appears on:MongoDBSpec)

SSLMode represents available sslmodes of mongodb. ref: https://docs.mongodb.com/manual/reference/program/mongod/#cmdoption-mongod-sslmode

Value Description

"allowSSL"

SSLModeAllowSSL represents allowSSL sslMode. It ensures that the connections between servers do not use TLS/SSL. For incoming connections, the server accepts both TLS/SSL and non-TLS/non-SSL.

"disabled"

SSLModeDisabled represents disabled sslMode. It ensures that the server does not use TLS/SSL.

"preferSSL"

SSLModePreferSSL represents preferSSL sslMode. It ensures that the connections between servers use TLS/SSL. For incoming connections, the server accepts both TLS/SSL and non-TLS/non-SSL.

"requireSSL"

SSLModeRequireSSL represents requiteSSL sslmode. It ensures that the server uses and accepts only TLS/SSL encrypted connections.
